How to install and use Grav CMS

Here I will explain how to install Grav CMS with composer.

I have bought another domain name! Now what? I've been wanting to try Grav CMS for a while, so lets try installing it...

Requirements:
You need:

- A web hosting account that allows SSH access

- A web hosting account that has Composer installed

- You need to make sure you have SSH access, and install Composer if you are using your own server.

Firstly, you need to make sure you have SSH access to your server. In my case, I have a premium account with Hostinger, which allows me SSH access.

Skip this section if you dont use Hostinger: I'm sure your hosting provider has instructions about where to find your SSH details. To find your SSH details in Hostinger, go to your control panel. Then click on your hosting account, (in my case 109-lyndhurst.uk) and click manage.

Now scroll down, till you get to the section 'Advanced', and click on 'SSH Access'. This will take you to another page, where you can enable SSH Access. So go ahead and do that... Click enable, and update. (I always think green is nicer than red...?)

Awesome! We are ready to roll. Next, you will need an SSH Client. I'm using Windows, so I have chosen MobaXterm. This is a nice SSH/SFTP/FTP client which I find easier to use than Putty etc. But you can use whatever you feel comfortable with.
Login to your server with the SSH details, ssh username@host -p 65002(note: Hostinger uses a different port for SSH, hence i have added the -p 65002. If your host uses the default port, there is no need to add that part) and if all goes well, you should be able to access the command line! Excellent.

We're in!

Okay, let's explore a little bit. On Hostinger, SSH takes you to a level up from the main folder. So if you type ls, you will see a folder called public_html.
-bash-4.2$ ls
DO_NOT_UPLOAD_HERE public_html

It is important to note that the server only shows files inside that folder, anything above public_html will not be shown.
Let's start installing Grav!
Firstly, enter the public_html folder, with cd public_html. Now type ls. Presuming it is a brand new account, there may be a default or example page in this folder.

It is also important to note that Composer won't install Grav if the folder already has content in it. This means that if you do have a default page, you will need to remove it.
This can be done by typing rm [filename], or if there are multiple files/folders, type rm -r *. This tells the server to recursively (-r) remove all content in that folder/subfolders (*). Now we have an empty folder! We can finally get to the gritty bits, really installing Grav CMS. We can simply install Grav by typing this short line. composer create-project getgrav/grav .. This tells composer to install Grav in the current folder (.). Composer will start its thing, and download all files and dependencies etc. It may take a while... relax, and catch up on some wildlife documentaries.

When Composer has finished, we still have a couple more things to do. When we install Grav with composer, it only installs the base system.
Currently, if you browse to the web page where you installed Grav, it will say you have no theme. So, we need to install a theme, and a few plugins to make managing the site easier.
Make sure you are in the root of the grav installation (where you installed grav) and type bin/gpm install antimatter -y into the command line. This tells grav to install the default Antimatter theme using the Grav Package Manager (GPM). Once that has finished, if you refresh your web page, it should show the theme. Woop!
If this command doesnt work, type ls and check to see if there is a folder called bin. If there isn't, then you need to enter the public_html folder again, or browse to where you installed Grav.

One other plugin to install is the admin plugin. This provides a nice backend interface to manage your site. Type bin/gpm install admin -y. Once that has finished, again refresh the page. This time it should show you a registration screen, where you can set up an admin user.
Fill in the fields with a username, password etc, and click Create User. The page will refresh and take you to an admin page.
Here you can now start customising your site,with the site name etc. You can also add pages via the pages menu. These will use the default theme installed at the moment. Click on Pages, and click on add in the top right corner.
Fill in the webpage name and folder. This is the path you will access the webpage on. For example, I have named mine 'test', so I access the page at example.com/test.
The bar down the left-hand side also has options to install other themes and plugins.
